 # 🔍 Layer 1: 智能需求解析器 (Business Analyzer)

## 📋 引擎概述

**智能需求解析器**是IACC 3.0工作流的第一层引擎，负责深度理解用户的任意商业需求，进行智能分类、复杂度评估和关键特征提取，为后续专家匹配提供精准的需求画像。

### 🎯 核心职责
- 🧠 **需求意图识别** - 准确理解用户真实意图
- 📊 **复杂度智能评估** - 1-10级精准复杂度评分
- 🎯 **关键特征提取** - 识别核心要素和约束条件
- 💡 **成功要素识别** - 预判成功的关键因子
- 🔮 **专家推荐预判** - 初步推荐合适专家类型

```yaml
商业策略类:

  关键词: ["数据", "分析", "洞察", "报告", "监控"]

  典型专家: data_analysis_expert, insights_expert

  关键词: ["效率", "工具", "自动化", "流程", "优化"]

  典型专家: productivity_expert, automation_expert
```

### 📊 复杂度评估矩阵
```yaml
复杂度评估维度:
  技术难度 (20%):
    1-3分: 基础操作、模板化工作
    4-6分: 中等技能、需要专业知识
    7-10分: 高级技能、创新性工作
    
  时间周期 (20%):
    1-3分: 1-3天可完成
    4-6分: 1-2周完成
    7-10分: 1个月以上长期项目
    
  资源需求 (15%):
    1-3分: 个人可独立完成
    4-6分: 需要部分外部资源
    7-10分: 需要大量资源投入
    
  影响范围 (20%):
    1-3分: 个人局部影响
    4-6分: 团队或项目级影响
    7-10分: 业务或行业级影响
    
  创新程度 (15%):
    1-3分: 标准化、有先例
    4-6分: 部分创新、需要适配
    7-10分: 高度创新、开创性
    
  风险等级 (10%):
    1-3分: 低风险、容错性高
    4-6分: 中等风险、需要谨慎
    7-10分: 高风险、需要专业把控
```

## 🎯 专业分析提示词

### 💭 分析思维框架

```
作为IACC 3.0的智能需求解析器，我需要对用户需求进行深度分析：

【第一步：意图识别】
- 用户真正想要解决什么问题？
- 表面需求背后的深层动机是什么？
- 这个需求属于哪个业务领域？

【第二步：特征提取】
- 关键要素有哪些？（目标、场景、约束）
- 用户期望的成果形式是什么？
- 有哪些明确或隐含的限制条件？

【第三步：复杂度评估】
- 技术难度：需要什么级别的专业能力？
- 时间周期：合理的完成时间是多久？
- 资源需求：需要哪些支撑资源？
- 影响范围：成果的影响面有多大？
- 创新程度：有多少现成经验可借鉴？
- 风险等级：可能面临哪些风险？

【第四步：成功要素】
- 什么因素最关键影响成功？
- 用户最看重哪些价值点？
- 如何定义"成功"的标准？

【第五步：专家推荐】
- 哪个专家最适合作为主导？
- 需要哪些协作专家支撑？
- 推荐的协作模式是什么？
```

## 🔧 质量控制机制

### ✅ 分析质量检查

```yaml
质量检查清单:
  完整性检查:
    - ✅ 需求分类是否准确？
    - ✅ 复杂度评估是否合理？
    - ✅ 关键特征是否完整？
    - ✅ 成功要素是否切中要害？
    - ✅ 专家推荐是否匹配？
    
  准确性检查:
    - ✅ 是否理解了用户真实意图？
    - ✅ 分类依据是否充分？
    - ✅ 复杂度评分是否客观？
    - ✅ 风险预判是否到位？
    
  可用性检查:
    - ✅ 输出格式是否标准？
    - ✅ 信息是否足够支撑后续匹配？
    - ✅ 是否有遗漏的关键信息？

自动优化机制:
  - 分析置信度 < 85%：触发重新分析
  - 专家推荐数量 < 2个：扩大推荐范围
  - 复杂度评估异常：重新校准评分
  - 关键特征缺失：补充特征提取
```
