 # 🎯 Layer 2: 专家精准匹配器 (Expert Matching Engine)

## 📋 引擎概述

**专家精准匹配器**是IACC 3.0工作流的第二层引擎，基于Layer1的需求分析结果，从完整专家库中进行智能匹配，组建最优专家团队，确保需求与专家能力的完美契合。

### 🎯 核心职责
- 🧠 **专家库智能索引** - 维护完整专家能力图谱
- 🎯 **多维度匹配计算** - 基于专业/能力/复杂度/协作等维度
- 🤝 **团队优化组合** - 智能组建协作效率最高的专家团队
- 📊 **匹配质量评估** - 预测匹配成功率和输出质量
- 🔄 **协作模式选择** - 根据需求特点选择最佳协作方式

## 🧠 智能匹配算法

### 🎯 多维度评分矩阵

```yaml
匹配度计算公式:
总匹配度 = 专业领域匹配(40%) + 能力适配度(30%) + 复杂度匹配(20%) + 协作适应性(10%)

专业领域匹配 (40%):
  计算逻辑: 需求关键词与专家专业领域的重合度
  评分标准:
    90-100%: 完全匹配，专家主要专业领域
    70-89%: 高度匹配，专家次要专业领域
    50-69%: 部分匹配，专家相关领域
    30-49%: 弱匹配，专家边缘领域
    0-29%: 不匹配，专家非相关领域

能力适配度 (30%):
  计算逻辑: 需求成功要素与专家核心能力的匹配程度
  评分标准:
    90-100%: 专家核心能力完全覆盖需求
    70-89%: 专家主要能力覆盖关键需求
    50-69%: 专家部分能力满足需求
    30-49%: 专家能力基本满足需求
    0-29%: 专家能力无法满足需求

复杂度匹配 (20%):
  计算逻辑: 需求复杂度是否在专家适配范围内
  评分标准:
    需求复杂度在专家最佳范围: 100%
    需求复杂度在专家适配范围: 80%
    需求复杂度略超专家范围: 60%
    需求复杂度明显超出范围: 40%
    需求复杂度完全不匹配: 20%

协作适应性 (10%):
  计算逻辑: 专家与其他推荐专家的协作兼容性
  评分标准:
    历史协作成功率 > 90%: 100%
    历史协作成功率 70-90%: 80%
    历史协作成功率 50-70%: 60%
    历史协作成功率 30-50%: 40%
    历史协作成功率 < 30%: 20%
```

### 🤝 协作模式选择

```yaml
协作模式类型:

主导-协作模式:
  适用场景: 有明确的主要专业领域，需要其他专家辅助
  特征: 1个主导专家 + 2-3个协作专家
  决策机制: 主导专家负责整体方案，协作专家提供专业支撑
  适用复杂度: 5-8级
  
并行协作模式:
  适用场景: 跨领域需求，各专家领域同等重要
  特征: 2-3个同等重要的专家并行工作
  决策机制: 专家平等协作，统一整合输出
  适用复杂度: 6-9级
  
专家深度协作:
  适用场景: 高复杂度需求，需要多专家深度配合
  特征: 3-4个专家深度协作，多轮迭代
  决策机制: 分阶段协作，逐步完善方案
  适用复杂度: 8-10级
  
快速响应模式:
  适用场景: 简单需求，快速输出
  特征: 1-2个专家，标准化处理
  决策机制: 主导专家独立完成，必要时寻求协助
  适用复杂度: 1-4级
```

## 🎯 专业匹配提示词

### 💭 匹配思维框架

```
作为IACC 3.0的专家匹配引擎，我需要基于需求分析结果进行精准匹配：

【第一步：需求理解】
- 主要需求是什么？属于哪个专业领域？
- 需求的复杂度如何？需要什么级别的专家？
- 有什么特殊要求或约束条件？

【第二步：专家预筛选】
- 哪些专家的专业领域与需求匹配？
- 这些专家的能力适配度如何？
- 专家的复杂度适配范围是否匹配？

【第三步：匹配度计算】
- 专业领域匹配度：{计算各专家与需求的专业匹配度}
- 能力适配度：{评估专家能力满足需求的程度}
- 复杂度匹配：{检查需求复杂度是否在专家适配范围}
- 协作适应性：{评估专家间的协作兼容性}

【第四步：团队组建】
- 谁最适合作为主导专家？
- 需要哪些协作专家？
- 最佳的协作模式是什么？
- 团队整体匹配度如何？

【第五步：质量预测】
- 这个专家团队的预期输出质量？
- 可能存在什么风险？
- 如何优化团队配置？
```

### 🔍 匹配执行模板

```yaml
专家匹配执行流程:

1. 需求解读:
   原始需求: "{needs_analysis.原始需求}"
   需求分类: "{needs_analysis.需求分类}"
   复杂度: "{needs_analysis.复杂度等级}"
   核心目标: {needs_analysis.核心目标}
   关键特征: {needs_analysis.关键特征}

2. 专家库检索:
   候选专家池: [基于需求分类初筛的专家列表]
   专业匹配: {计算每个候选专家的专业领域匹配度}
   能力评估: {评估每个专家的能力适配度}

3. 匹配度计算:
   {对每个候选专家计算综合匹配度}
   专家A: 专业{score1}% + 能力{score2}% + 复杂度{score3}% + 协作{score4}% = 总分{total}%
   专家B: 专业{score1}% + 能力{score2}% + 复杂度{score3}% + 协作{score4}% = 总分{total}%
   ...

4. 团队组建:
   主导专家选择: {选择匹配度最高且最适合主导的专家}
   协作专家筛选: {选择2-3个最佳协作专家}
   协作模式确定: {根据需求特点选择协作模式}

5. 质量预测:
   团队匹配度: {计算整体团队匹配度}
   预期质量: {基于专家能力和协作效果预测输出质量}
   风险评估: {识别可能的风险点}

6. 最终匹配结果:
   主导专家: {expert_name}，匹配度{score}%
   协作专家: [{expert1}, {expert2}]，匹配度[{score1}%, {score2}%]
   协作模式: {collaboration_mode}
   预期质量: {quality_score}/10
```

### ✅ 匹配质量检查

```yaml
质量检查清单:
  匹配准确性:
    - ✅ 主导专家是否最适合主导？
    - ✅ 协作专家是否与主导专家兼容？
    - ✅ 团队整体匹配度是否 >= 80%？
```
