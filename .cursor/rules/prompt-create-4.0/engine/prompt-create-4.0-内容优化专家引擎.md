# 🎯 Prompt-Create-4.0 内容优化专家引擎

## 🚀 核心使命
> **基于AI-Write-2.0正文结构专家系统，为提示词创建提供结构式+套路式内容优化建议**

## 🎛️ 双引擎架构

### 🏗️ 结构式优化引擎
```yaml
金字塔结构优化:
  顶层核心:
    - 核心价值: "这个提示词的核心价值是..."
    - 主要功能: "主要解决...问题"
    - 适用场景: "特别适合...场景"
    
  中层支撑:
    - 功能模块: "包含...等核心功能"
    - 技术特点: "采用...技术/方法"
    - 使用优势: "相比传统方法，具有...优势"
    
  底层细节:
    - 具体步骤: "使用时只需...几个步骤"
    - 操作指南: "具体操作是..."
    - 注意事项: "需要注意..."

递进式价值展示:
  第一层 - 基础价值:
    - 直接效果: "能够直接帮你..."
    - 时间节约: "节省...时间"
    - 效率提升: "提升...效率"
    
  第二层 - 进阶价值:
    - 质量保证: "确保...质量"
    - 专业水准: "达到...专业水准"
    - 个性化: "根据...需求定制"
    
  第三层 - 深层价值:
    - 能力提升: "培养...能力"
    - 思维转变: "改变...思维方式"
    - 长期收益: "带来...长期收益"

对比强化结构:
  横向对比:
    - 与传统方法对比: "传统方法vs这个提示词"
    - 与同类工具对比: "其他工具vs这个提示词"
    - 与人工操作对比: "人工操作vs AI辅助"
    
  纵向对比:
    - 使用前后对比: "使用前...，使用后..."
    - 能力层次对比: "新手水平vs专家水平"
    - 成本效益对比: "投入成本vs获得收益"
    
  维度对比:
    - 速度对比: "处理速度提升...倍"
    - 质量对比: "质量提升...等级"
    - 成本对比: "成本降低...百分比"

场景化应用结构:
  核心场景:
    - 主要应用: "最常用于...场景"
    - 典型用户: "特别适合...用户"
    - 使用频率: "通常...情况下使用"
    
  扩展场景:
    - 衍生应用: "还可以用于..."
    - 特殊场景: "在...特殊情况下"
    - 创新应用: "创新性地用于..."
    
  场景匹配:
    - 场景识别: "如何判断是否适用"
    - 参数调整: "不同场景的参数设置"
    - 效果预期: "在不同场景下的效果"
```

## 🔍 5维度质量评估

### 📊 评估框架
```yaml
1. 逻辑清晰度 (0-100分):
   - 结构完整: 是否有清晰的逻辑结构
   - 层次分明: 是否层次清晰，递进合理
   - 论证充分: 是否有充分的论证支撑
   
2. 价值传达度 (0-100分):
   - 价值明确: 是否清楚传达价值主张
   - 收益具体: 是否有具体的收益描述
   - 说服力强: 是否具有说服力
   
3. 实用性指数 (0-100分):
   - 应用清晰: 是否清楚说明应用方法
   - 场景贴合: 是否贴合实际使用场景
   - 可操作性: 是否具有强可操作性
   
4. 平台适配度 (0-100分):
   - 内容长度: 是否符合平台内容长度特点
   - 表达风格: 是否符合平台用户偏好
   - 互动性: 是否具有良好的互动性
   
5. 说服力指数 (0-100分):
   - 证据支撑: 是否有充分的证据支撑
   - 情感共鸣: 是否能引起情感共鸣
   - 行动驱动: 是否能驱动用户行动
```

### 🎯 评分标准
```yaml
优秀 (90-100分):
  - 逻辑结构完美，层次清晰
  - 价值传达明确，收益具体
  - 高度实用，可操作性强
  - 完美适配平台特色
  - 说服力极强，证据充分
  
良好 (80-89分):
  - 逻辑结构较好，基本清晰
  - 价值传达较为明确
  - 实用性较强，有一定指导性
  - 基本适配平台特色
  - 有一定说服力
  
及格 (70-79分):
  - 基本逻辑结构，不够清晰
  - 价值传达一般
  - 实用性一般
  - 基本符合平台要求
  - 说服力一般
  
需改进 (60-69分):
  - 逻辑结构不清晰
  - 价值传达不明确
  - 实用性不强
  - 平台适配度不够
  - 说服力不足
  
不合格 (<60分):
  - 缺乏逻辑结构
  - 价值传达模糊
  - 缺乏实用性
  - 不符合平台特色
  - 没有说服力
```

## 🎨 建议整合系统

### 🔄 优化建议生成流程
```yaml
第一步: 内容分析
  - 当前内容结构分析
  - 价值传达效果评估
  - 逻辑流程梳理
  - 平台特征匹配分析
  
第二步: 结构式优化建议
  - 金字塔结构重构: 基于核心价值重新组织内容结构
  - 递进式价值优化: 构建价值递进的展示逻辑
  - 对比强化策略: 通过对比突出优势和价值
  - 场景化应用改写: 提供具体的应用场景和案例
  
第三步: 套路式优化建议
  - 问题-方案-效果链: 构建问题解决的完整链条
  - 特征-好处-利益链: 将特征转化为用户利益
  - 权威-证据-信任链: 建立权威性和可信度
  - 稀缺-紧迫-行动链: 驱动用户采取行动
  
第四步: 平台自适应调整
  - 微信公众号: 逻辑性、专业性强化
  - 小红书: 体验性、互动性优化
  - 内容长度: 平台特色长度适配
  - 表达风格: 平台用户习惯调整
  
第五步: 综合评分与排序
  - 5维度评分: 逻辑、价值、实用性、适配度、说服力
  - 优化效果预测: 预测优化后的效果提升
  - 实施难度评估: 评估实施的难易程度
  - 优先级排序: 按效果和难度排序
```
